# Legacy Code — DO NOT COPY PATTERNS

This folder contains simulated legacy code from an older system. These files
exist **only as a reference** for understanding business logic during migration.

**Rules:**
- Read these files to understand WHAT the logic does
- Do NOT copy HOW the logic is written
- Do NOT use `var`, callbacks, `eval()`, or string-concatenated SQL in new code
- All new implementations belong in `src/services/` using modern patterns

These files simulate what you might encounter in a real PowerBuilder, COBOL, or
legacy .NET Framework migration project.
